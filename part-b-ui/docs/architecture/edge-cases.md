# 예외/엣지 케이스

고려한 예외 상황과 UI 대응 방안을 기술합니다.
형식: 상황 → UI 대응 방안

---

## 입력값 관련

| 상황 | UI 대응 |
|------|---------|
| 발신번호 형식 오류 (포커스 이탈 시) | 필드 빨간 테두리 + "올바른 발신번호 형식이 아닙니다. (예: 010-1234-5678)" 인라인 에러. [저장] 비활성화 |
| 필수 필드 미입력 상태에서 저장 시도 | 빈 필드 빨간 테두리 + "필수 항목입니다" 인라인 에러 |
| API Key에 앞뒤 공백 포함 붙여넣기 | onBlur 시 자동 trim 후 재검증 |
| 마스킹 입력 필드 클릭 후 아무것도 입력 안 하고 이탈 | 마스킹 값 복원 (변경 없음으로 처리) |

---

## 연결 테스트 관련

| 상황 | UI 대응 |
|------|---------|
| `AUTHENTICATION_FAILED` / `UNAUTHORIZED` | "API Key가 올바르지 않습니다." + API Key 필드로 포커스 이동 |
| `INVALID_SIGNATURE` | "Secret Key가 올바르지 않습니다." + Secret Key 필드로 포커스 이동 |
| `INVALID_SENDER` / `INVALID_PHONE_NUMBER` | "등록되지 않은 발신번호입니다. 업체에서 발신번호 등록 여부를 확인해주세요." + 발신번호 필드로 포커스 이동 |
| `QUOTA_EXCEEDED` / `RATE_LIMIT_EXCEEDED` | "요청 한도를 초과했습니다. 잠시 후 다시 시도해주세요." + [재시도] 버튼 |
| `SERVICE_UNAVAILABLE` / `INTERNAL_ERROR` | "업체 서버에 일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요." + [재시도] 버튼 |
| 테스트 성공 후 값 수정 | 성공 배너 제거, [저장] 버튼 비활성화 → 재테스트 유도 |
| 테스트 중 [연결 테스트] 재클릭 | 버튼 비활성화로 중복 요청 차단 |

---

## 저장 관련

| 상황 | UI 대응 |
|------|---------|
| 저장 중 [저장] 재클릭 | 버튼 비활성화 + 스피너로 중복 요청 차단 |
| 저장 API 4xx 응답 | "입력값을 다시 확인해주세요." 에러 토스트 |
| 저장 API 5xx 응답 | "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요." 에러 토스트 |
| 네트워크 오류 / 타임아웃 | "네트워크를 확인해주세요." 에러 토스트 |
| 동일 업체 동일 Credential 재저장 | 정상 처리 (Credential 갱신 목적으로 허용, 경고 없음) |

---

## 업체 전환 관련

| 상황 | UI 대응 |
|------|---------|
| 업체 전환 시 입력 중이던 값 | 전체 초기화 (이전 업체 값 잔류 방지) |
| 업체 전환 후 다시 원래 업체로 | 초기화된 상태 유지 (원래 값 복원 안 함) — 의도적 변경으로 간주 |

---

## 페이지 상태 관련

| 상황 | UI 대응 |
|------|---------|
| 미저장 변경 상태에서 페이지 이탈 | "저장되지 않은 변경사항이 있습니다. 페이지를 떠나시겠습니까?" 브라우저 확인 다이얼로그 |
| 페이지 진입 시 GET API 실패 | "설정을 불러오는 데 실패했습니다. 새로고침 해주세요." 에러 배너 |
| 권한 없음 (403) | "접근 권한이 없습니다." 전체 화면 전환 |
| 설정이 없는 조직 (404) | 노란 경고 배너 표시 후 편집 영역 정상 노출 (신규 설정 가능 상태) |
