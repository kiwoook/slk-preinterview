# 화면 흐름

사용자가 업체 설정을 완료하기까지의 행위 순서를 단계별로 기술합니다.

---

## 전체 흐름 개요

```
[페이지 진입]
     ↓
[현재 설정 조회 (GET API)]
     ↓
 ┌───┴────────────────────────┐
 │ 설정 있음                  │ 설정 없음
 │ 현재 설정 카드 표시        │ 경고 배너 표시
 └───┬────────────────────────┘
     ↓
[업체 선택 (라디오 버튼)]
     ↓
[Credential 입력 (업체별 폼)]
     ↓
[연결 테스트] ── 실패 → 원인별 에러 메시지 → 수정 후 재시도
     ↓ 성공
[저장 클릭]
     ↓
 ┌───┴──────────────────────────────┐
 │ 업체 변경                        │ 동일 업체 (Credential만 변경)
 │ 확인 다이얼로그 표시             │ 바로 PUT API 호출
 └───┬──────────────────────────────┘
     ↓ 확인
[PUT API 호출]
     ↓
 ┌───┴───────────────────────┐
 │ 성공                      │ 실패
 │ 토스트 + 현재 설정 카드 갱신 │ 에러 토스트
 └───────────────────────────┘
```

---

## 단계별 상세

### 1단계. 페이지 진입

- [`GET /organizations/{orgId}/message-provider`](../../../part-a-server/docs/api/api-spec.md#get-organizationsorgidmessage-provider) 호출
- **설정 있음**: 상단 "현재 적용 중인 설정" 카드에 업체명, 마스킹된 Credential, 마지막 변경일시 표시
- **설정 없음** (404): 노란색 경고 배너 "아직 업체가 설정되지 않았습니다. 아래에서 설정을 완료해주세요."
- 편집 영역의 업체 라디오 버튼은 현재 설정 업체로 기본 선택

### 2단계. 업체 선택

- 라디오 버튼으로 업체 선택 (클라우드메시지 / 센드톡)
- 업체 전환 시: 이전 업체 Credential 입력값 전체 초기화 후 새 업체 폼 표시
- 현재 설정과 동일한 업체 선택 시: 기존 Credential을 마스킹 상태로 채워둠

### 3단계. Credential 입력

- 업체별 필드 표시 ([프로토타입 명세 참고](../prototype/prototype.md#업체별-credential-입력-필드))
- 발신번호: 숫자 입력 시 하이픈 자동 삽입, 포커스 이탈 시 형식 검증
- 마스킹 입력 필드: 기존 값 마스킹 표시 → 클릭 시 초기화 후 재입력
- 모든 필수 필드 채워지면 [저장] 버튼 활성화

### 4단계. 연결 테스트

- [연결 테스트] 클릭 → [`POST /organizations/{orgId}/message-provider/test`](../../../part-a-server/docs/api/api-spec.md#post-organizationsorgidmessage-providertest)
- 테스트 중: 버튼 스피너, [저장] 비활성화
- 성공: 초록 인라인 배너 "✓ 연결 성공. 저장해주세요."
- 실패: `providerErrorCode` 기준 원인별 메시지 + 해당 필드로 포커스 이동
- 값 수정 시: 테스트 결과 초기화 (낡은 성공 결과로 저장 방지)

### 5단계. 저장

- **업체가 변경되는 경우**: 확인 다이얼로그 표시
  - "진행 중인 발송은 기존 업체로 완료됩니다. 변경은 다음 발송부터 적용됩니다."
  - [취소] / [변경 확인]
- **동일 업체 Credential만 변경**: 확인 다이얼로그 없이 바로 [`PUT /organizations/{orgId}/message-provider`](../../../part-a-server/docs/api/api-spec.md#put-organizationsorgidmessage-provider) 호출
- 저장 중: [저장] 버튼 비활성화 + 스피너 (중복 요청 방지)

### 6단계. 저장 결과

- **성공**: 상단 토스트 "설정이 저장되었습니다." (3초 자동 닫힘) + 현재 설정 카드 즉시 갱신
- **실패 (4xx)**: "입력값을 다시 확인해주세요."
- **실패 (5xx)**: "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요."

## 이탈 방지

- 입력값이 변경된 미저장 상태에서 페이지 이탈 시:
  ```
  저장되지 않은 변경사항이 있습니다. 페이지를 떠나시겠습니까?
  [취소]  [떠나기]
  ```
